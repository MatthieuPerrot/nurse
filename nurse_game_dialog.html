Dialogs in The Nurse ProjectÂ¶

Note

22/05/2011

The main() function may instanciate a nurse.game.dialog.DialogContext which is added to a list of contexts managed by the nurse.context.ContextManager. The nurse.game.DialogContext __init__() function instanciates various sprites (such as nurse.sprite.UniformLayer), a nurse.sprite.Dialog (nurse.sprite.Sprite (nurse.state_machine.StateMachine) ) and a nurse.game.dialog.DialogListener.

As a nurse.sprite.Sprite and therefore a nurse.state_machine.StateMachine, the nurse.sprite.Dialog has a member states which is a list of nurse.game.dialog.DialogState (nurse.state_machine.State). The nurse.sprite.Dialog.update() function actually just calls the nurse.game.dialog.DialogState.update() function of the nurse.game.dialog.DialogState defined by the member _current_state of the nurse.sprite.Dialog.

For each nurse.sprite.Sprite and in particular for the nurse.sprite.Dialog, the method nurse.state_machine.StateMachine.start() is called, which initializes the member _current_state with _initial_state, should it be previously defined by the method nurse.state_machine.StateMachine.set_initial_state() or by a default instance created in that same nurse.state_machine.StateMachine.start() function. Then the nurse.context.ContextManager updates its nurse.context.Context, or more specifically each nurse.context.Context has its nurse.state_machine.StateMachine updated. With regard to this, the nurse.game.dialog.DialogContext updates its sprites, therefore the nurse.sprite.Dialog and its current nurse.game.dialog.DialogState.
